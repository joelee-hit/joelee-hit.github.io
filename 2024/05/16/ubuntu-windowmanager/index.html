ubuntu_windowmanager

本文最后更新于：2024年5月16日 晚上

一直期待在ubuntu 22.04 上实现平铺式窗口，所以有了此文章，记录我的探索。经过对比，选择轻量级窗口管理器dwm。在动手之前，先观看B站UP对dwm的安装和配置，还可以看ubuntu配置dwm。感觉dwm非常符合我对平铺式窗口管理器的想象，就是看到的博主都是用arch linux的，没有使用ubuntu实践过的。suckless极简桌面。

dwm

首先，下载dwm的源代码，据说代码非常小：

README

查看README文件：

首先，Xlib头文件一般包含在开发包libx11-dev里，ubuntu22.04中的版本是libx11-dev/jammy-updates,jammy-security,now 2:1.7.5-1ubuntu0.3 amd64，直接用apt安装即可，安装后可以在/usr/include/X11/下找到相应头文件。

其次，config.mk是dwm源代码安装时的配置文件，这样编译后就会安装到相应文件夹（更加轻量、速度更快）。

首先，exec dwm告诉xinit进程执行dwm作为X服务器的最后一个动作，即dwm将成为你的窗口管理器。DISPLAY=foo.bar:1 exec dwm用于在名为foo.bar的主机上的第二个显示器（:1）运行dwm。while xsetroot ... exec dwm用于在dwm的状态栏自定义信息，比如日期和系统运行时间。

其次，dwm的配置不是通过图形界面或单独的配置文件完成的，而是需要直接修改源代码中的config.h文件。这意味着如果你想调整dwm的行为，比如键绑定、颜色方案或是布局管理等，你需要编辑这个C头文件，然后重新编译dwm的源代码。这种配置方式虽然较为繁琐，但为用户提供极高的定制自由度。

PREFIX, MANPREFIX是dwm的安装位置和man帮助文件所处位置，可以不用改。

X11INC, X11LIB这里是过时的/usr/X11R6，应该改成X11INC = /usr/include/X11和X11LIB = /usr/lib/x86_64-linux-gnu。/usr/lib/x86_64-linux-gnu下有libX11.so等文件。libx11-dev。

Xinerama是X Window系统的一个扩展，它允许应用程序跨越多个物理显示器并将其视作单个逻辑屏幕，从而实现多屏拼接显示，头文件在包libxinerama-dev里。平常的笔记本可以不要。况且现代的显示器通常用XRandR？

FREETYPELIBS和FREETYPEINC不用改变。FreeType是一个开源的、高质量、可移植的字体引擎库，它被设计为轻量级、高效并支持高度定制。libfontconfig-dev包含开发时需要的头文件和库，FontConfig是一个字体配置和定制库；libxft-dev: Xft库是基于FreeType库的，专门用于在X Window系统中渲染字体。

可以用pkg-config查询库和依赖文件，比如pkg-config --cflags --libs x11，查询X11的库、依赖、cflags。

再看makefile文件：

  1. 注释行，简要描述项目名称及版权和许可详情所在文件。
  2. 包含config.mk文件，该文件通常包含编译时的配置信息，比如编译器选项、安装路径等。
  3. 定义了需要编译的源文件列表。
  4. 将源文件列表转换为对应的对象文件列表。Makefile的模式替换功能，${SRC:.c=.o}表示将SRC变量中每个源文件名的.c后缀替换为.o，相当于OBJ = drw.o dwm.o util.o。
  5. all目标：默认规则，用于构建最终的可执行文件dwm。
  6. 规则模板，用于将任何.c文件编译为.o（目标文件）。使用${CC}编译器，并应用${CFLAGS}编译选项。$<表示第一个依赖项。
  7. ${OBJ}: config.h config.mk指出所有对象文件依赖于config.h和config.mk，这意味着每次这两个文件改变时，所有对象文件都需要重新编译。
  8. config.h的生成规则，简单地从config.def.h复制而来。$@是目标文件名，也就是config.h。
  9. dwm可执行文件的链接规则，将所有对象文件链接成最终的二进制文件。${OBJ}表示该规则对应的依赖。
  10. clean目标：删除编译过程中产生的中间文件和可执行文件。
  11. dist目标：依赖于clean规则，因此会先执行clean，再创建项目的发布包，包括创建版本目录、复制必要文件、打包并压缩。
  12. install目标：安装编译好的dwm到系统中指定的位置（由${PREFIX}和${MANPREFIX}定义），并设置适当的权限。${DESTDIR}没有设置时通常为空字符串。man1 存放的是第1节的手册页，
  13. uninstall目标：从系统中卸载dwm及相关手册页。
  14. .PHONY指令，声明后面列出的目标为伪目标，意味着它们不会对应到文件，而总是执行指定的命令。

这个不需要任何改动，不过需要理解。

config.def.h

由makefile文件可知，config.h是直接由config.def.h复制而来，所以我们需要关注config.def.h文件：

设置外观、布局、快捷键、标签、窗口规则、以及按钮行为。说明我们还需要安装dmenu（一个菜单图形界面）、st（一个轻量级终端模拟器）。

外观

  • borderpx: 窗口边框像素宽度。
  • snap: 窗口对齐的像素点。当你尝试将一个窗口的边缘靠近屏幕边缘或另一个窗口时，一旦距离小于32像素，窗口会自动贴紧边缘或与相邻窗口对齐，而不是停留在一个随机的距离上。
  • showbar: 是否显示状态栏，1为显示。
  • topbar: 状态栏位置，1为顶部。
  • fonts: 字体设置。
  • dmenufont: dmenu使用的字体。dmenu是通常与dwm一起使用的可应用于脚本的图形选择界面程序，详细教程。dmenu接受标准输入（可以是echo，也可以是任何其他脚本的输出），用换行分隔，打开一个图形菜单，将前n项显示出来；用户可以输入并搜索，用户输入回车后，dmenu返回用户选择的菜单项；或者用户按Esc，dmenu什么也不返回；或者用户输入一个和任何菜单项都不匹配的字符串，dmenu返回这个字符串。
  • col_gray1-4, col_cyan: 颜色定义。
  • colors: 配置正常和选中状态下的前景、背景和边框颜色。

标签

  • tags[]: 定义了9个工作标签，每个标签对应一个数字。

规则(rules)

  • 定义了一些特定应用程序的窗口行为，例如Gimp总是浮动，Firefox窗口被分配到特定的标签上。

  • xprop是一个命令行工具，可以用来查看X窗口的属性（输入xprop后点击窗口），其中WM_CLASS和WM_NAME是两个关键属性，分别用于标识窗口的类（通常包括实例名和类别名）和窗口的标题。比如xprop -name "typora" WM_CLASS WM_NAME会返回：

  • class：匹配窗口的类名（窗口的WM_CLASS属性的第一部分，即实例名）。

  • instance：匹配窗口的实例名（窗口的WM_CLASS属性的第二部分）。这里设置为NULL，意味着不关心实例名，只匹配基于类名。

  • title：匹配窗口的标题（WM_NAME属性）。同样设置为NULL，表示不基于窗口标题进行匹配。

  • tags mask：定义了窗口应归属的标签掩码。例如，1 << 8（即256）意味着窗口将标记到第9个标签上（标签索引从0开始计数）；0表示不显式分配标签。

  • isfloating：布尔值，决定窗口是否浮动。1表示浮动（不遵循平铺布局，可自由调整大小和位置），0表示窗口遵循布局管理。

  • monitor：指定窗口应出现在哪个显示器上。-1表示自动选择，其他正整数指定了具体的显示器编号。

布局(layouts)

  • mfact: 主区域占屏幕的比例。
  • nmaster: 主区域中窗口数量。
  • layouts: 支持的布局模式，包括平铺(tile)、浮动(NULL，即默认的无布局管理，窗口自由浮动)和单窗口全屏(monocle)。
  • { "[]=", tile }：这是默认布局，符号为"[]=". 当选择此布局时，调用tile函数来排列窗口。这种布局会按照mfact和nmaster的设定，将一个或多个窗口放在主区域，并将剩余的窗口平铺在栈中区域。符号通常会在状态栏显示，让用户知道当前所选的布局模式。
  • { "><>", NULL }：这个布局没有对应的排列函数，意味着它代表浮动布局。在这种模式下，窗口会忽略平铺规则，可以自由调整大小和位置，如同在传统的窗口管理模式中一样。符号为"><>"。
  • { "[M]", monocle }：这是一种名为“monocle”的布局，符号为"[M]"。选择此布局时，所有窗口都会依次全屏显示，每次只显示一个窗口，适合需要专注于单一任务的场景。monocle函数负责实现这一布局逻辑。

键定义(keys)

  • 定义了各种键盘快捷键，如打开dmenu、切换标签、调整窗口布局、控制窗口堆栈、聚焦窗口、移动窗口、调整主区域大小等。XK_F4表示MODKEY(alt键)+F4

    • #define MODKEY Mod1Mask：这一行定义了MODKEY作为一个宏，它代表了键盘上的一个修饰键，通常是Alt键（Mod1）。在DWM中，这个修饰键通常与其他键组合使用，执行特定的操作。
    • TAGKEYS(KEY,TAG)宏：这个宏是用来方便地定义与标签操作相关的四个快捷键。它接收两个参数，KEY代表你按下要绑定的按键（如A, B, C等），而TAG则是对应标签的索引（基于之前定义的标签数组）。宏展开后会生成四个键绑定规则，分别对应于查看标签、切换查看标签、将窗口移动到标签、以及切换窗口的标签状态。
    • { MODKEY, KEY, view, {.ui = 1 << TAG} }：按下MODKEY（通常是Alt）加上KEY，会将视图切换到对应标签（TAG）的工作区。.ui = 1 << TAG计算出对应标签的位掩码，用于内部标记。
    • { MODKEY|ControlMask, KEY, toggleview, {.ui = 1 << TAG} }：同时按下MODKEY+Ctrl+KEY，会切换当前工作区是否显示指定标签（TAG）的内容。如果当前工作区已显示该标签，则隐藏；如果未显示，则显示。
    • { MODKEY|ShiftMask, KEY, tag, {.ui = 1 << TAG} }：按下MODKEY（Alt）+Shift+KEY，会将当前窗口移动到指定的标签（TAG）工作区。
    • { MODKEY|ControlMask|ShiftMask, KEY, toggletag, {.ui = 1 << TAG} }：同时按下MODKEY+Ctrl+Shift+KEY，会切换当前窗口是否属于指定的标签（TAG）。如果窗口已经属于该标签，则取消；如果不在，则添加。
    • #define SHCMD(cmd): 这是一个宏定义的开始，其中cmd是一个占位符，代表你想要执行的shell命令字符串。
    • { .v = (const char*[]){ "/bin/sh", "-c", cmd, NULL } }: 宏展开后生成的结构体初始化表达式。这个结构体通常用于向DWM指定一个可以执行shell命令的操作。具体来说：
      • .v: 这是一个结构体的成员，指向一个字符指针数组。在DWM中，这样的结构体常用于定义按键绑定时执行的动作。
      • (const char*[]): 定义了一个字符指针数组，数组中的每个元素都是一个指向字符串的指针。
      • "/bin/sh": 数组的第一个元素是/bin/sh，它指向Shell的路径。Shell是一个命令行解释器，可以执行命令。
      • "-c": 数组的第二个元素是-c，这是一个选项，告诉Shell接下来的参数是一个要执行的命令字符串。
      • cmd: 数组的第三个元素是宏参数cmd，替换时会填入实际的shell命令字符串。
      • NULL: 数组的末尾通常用NULL来表示结束，这是C语言中常见的做法。

命令(commands)

  • dmenumon: 控制dmenu在哪个显示器上显示。dmenucmd: dmenu的命令行参数。termcmd: 默认终端模拟器启动命令。

    • char dmenumon[2] = "0";：定义了一个字符数组，用来存储传递给dmenu的监视器编号。默认值为"0"，意味着dmenu将在主监视器上显示。
    • static const char *dmenucmd[]：定义了启动dmenu的命令及其参数。dmenu是一个小巧的动态菜单工具，常用于快速启动程序。参数包括：
      • -m, dmenumon: 指定在哪个显示器上显示dmenu。
      • -fn, dmenufont: 指定dmenu使用的字体。
      • -nb, -nf, -sb, -sf: 分别设置背景色、前景色、选择条的背景色和前景色，颜色值由其他变量如col_gray1等定义。
    • static const char *termcmd[]：定义了启动终端（在这里是st，一个简单终端模拟器）的命令。

  • 这部分定义了一系列键盘快捷键，用于控制窗口管理和执行特定操作。快捷键由四个主要部分组成：修饰键（如MODKEY，一般映射为Alt或Super键）、普通键、功能（如spawn、togglebar）、以及该功能的具体参数。

    例如：

    • { MODKEY, XK_p, spawn, {.v = dmenucmd } }：当按下MODKEY（假设是Alt）+ P时，执行dmenucmd数组定义的命令，即启动dmenu。
    • { MODKEY|ShiftMask, XK_Return, spawn, {.v = termcmd } }：按下MODKEY+ Shift + Return时，启动终端模拟器。
    • { MODKEY, XK_b, togglebar, {0} }：按下MODKEY+ B隐藏或显示窗口标题栏。
    • 其他键绑定分别控制着聚焦、布局调整、窗口浮动、视图切换、标签页管理、退出DWM等功能。
    • spawn在此处的作用是作为桥梁，将键盘事件转换为系统命令的执行，是实现快捷启动应用或其他系统操作的关键机制。

  • 按下MODKEY（通常是Alt键）+ p，会启动dmenu，用户可以在弹出的菜单中输入命令或选择程序进行执行。

    按下MODKEY+ Shift + 回车，会打开一个新的终端窗口（使用termcmd定义的终端模拟器，如suck或更正为st）。

    按下MODKEY+ b，可以隐藏或显示窗口管理器的状态栏。

    MODKEY+ j 下移焦点到下一个窗口，MODKEY+ k 上移焦点到上一个窗口。

    MODKEY+ i 增加主区域窗口数量，MODKEY+ d 减少主区域窗口数量。

    MODKEY+ h 缩小主区域宽度，MODKEY+ l 增大主区域宽度。

    按下MODKEY+ 回车，可以使当前窗口最大化（或恢复原大小）。

    MODKEY+ Tab 切换到下一个标签页。

    MODKEY+ Shift + c 关闭当前焦点所在的窗口。

    分别按下MODKEY+ t、f、m，会切换到预设的三种布局：平铺、浮动、或单窗口全屏模式。

    MODKEY+ 空格 快速循环切换到下一个布局。

    MODKEY+ Shift + 空格 切换当前窗口的浮动状态。

    MODKEY+ 0 查看所有标签页内容。MODKEY+ Shift + 0 将当前窗口分配到所有标签页。

    MODKEY+ , 切换到左边的显示器，MODKEY+ . 切换到右边的显示器。

    MODKEY+ Shift + , 将当前窗口移动到左边的显示器，MODKEY+ Shift + . 移动到右边的显示器。

    分别按下MODKEY+ 数字键1到9，可以快速切换到对应的标签页。

    MODKEY+ Shift + q 退出DWM窗口管理器。

按钮(buttons)

  • 定义了鼠标点击事件与动作的映射，如在状态栏点击切换布局、在窗口标题上双击放大窗口等。
    1. { ClkLtSymbol, 0, Button1, setlayout, {0} }:
      • ClkLtSymbol: 操作发生于显示当前布局指示符的位置。
      • 0: 不需要额外的事件掩码。
      • Button1: 鼠标左键。
      • setlayout: 动作是切换布局。
      • {0}: 传递给setlayout函数的参数为空，意味着切换到下一个布局。
    2. { ClkLtSymbol, 0, Button3, setlayout, {.v = &layouts[2]} }:
      • 与上一条类似，但当点击鼠标右键时，直接切换到数组layouts中的第三个布局（数组索引从0开始，所以这是第3个布局）。
    3. { ClkWinTitle, 0, Button2, zoom, {0} }:
      • 在窗口标题上点击鼠标中键会执行zoom操作，通常是使窗口最大化或恢复原大小。
    4. { ClkStatusText, 0, Button2, spawn, {.v = termcmd } }:
      • 点击状态文本（可能显示时间、系统信息等）的鼠标中键会启动一个终端（由termcmd变量定义）。
    5. { ClkClientWin, MODKEY, Button1, movemouse, {0} } 至 { ClkClientWin, MODKEY, Button3, resizemouse, {0} }:
      • 当按下MODKEY（比如Alt键）并点击客户端窗口的不同按钮时，可以移动（Button1）或调整窗口大小（Button3）。
    6. { ClkTagBar, 0, Button1, view, {0} } 至 { ClkTagBar, MODKEY, Button3, toggletag, {0} }:
      • 在标签栏上操作，无修饰键点击左右键分别用于查看（view）或切换视图（toggleview）指定的工作区；按下MODKEY时点击左右键则是给工作区添加（tag）或移除（toggletag）标签。
