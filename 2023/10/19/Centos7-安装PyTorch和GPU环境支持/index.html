Pytorch、CUDA、cudnn、NVIDIA驱动之间的兼容性

首先，可以明确，CUDA(CUDA Toolkit)、PyTorch、cudnn之间有一定的匹配关系，不能随便安装。

CUDA与驱动

从CUDA官网可以看到，CUDA与驱动的兼容性如下：

值得注意的是，在CUDA Toolkit的安装程序中附加了NVIDIA驱动的安装，但是官方并不建议，官方建议到NVIDIA驱动选择界面根据自己的Tesla GPU来选择驱动。

也就是说，NVIDIA驱动的版本是越新越好。

另外，可以在这里查看自己显卡的算力。这是A100显卡的。

关于CUDA向后兼容性的理解，可以看看这篇文章。

cudnn与CUDA

cuDNN是用于深度神经网络的GPU加速库，相当于是对CUDA的补充。可以在cuDNN官网选择与已安装的CUDA版本对应的下载。

值得注意的是，tensorflow-gpu对cuDNN的版本也有一定限制，可以在这里查看。就是不知道PyTorch有没有？！在这里可以看到CUDA只要满足PyTorch要求的最低版本即可，越新越好。

To install PyTorch via Anaconda, and you do have a CUDA-capable system, in the above selector, choose OS: Linux, Package: Conda and the CUDA version suited to your machine. Often, the latest CUDA version is better. Then, run the command that is presented to you.

PyTorch与CUDA、Python

在PyTorch官网上，可以看到Pytorch对于不同版本的CUDA有不同的安装指令。

在PyTorch的get-started界面可以看到PyTorch对OS发行版、Python版本($$3.8)都有一定要求，也有验证是否能使用GPU的方法。

总结，不同版本的PyTorch要求不同的最低版本的CUDA和不同版本的Python，而CUDA又决定该选择什么版本的cuDNN。
